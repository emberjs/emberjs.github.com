Roadmap

If you were ever curious about what lies ahead for Ember.js, this page should provide the information you're looking for.

Here we will track the progress of the various initiatives that continuously shape Ember.js, from their birth as RFCs, through Feature Flags and eventually into a Release.

canary

#186
Tracking unique history locations
Store a unique identifier along with each entry in the history. Useful for data associated with each page visit, rather than the page as a whole (such as scroll position).

RFC merged

#176
JavaScript Modules API
Introduces a new set of module groupings and exposes them as importable ES2015 packages.

This also paves the way for automated tree-shaking - automatically stripping out framework and addon code never used by your app.

#143
Module unification
Create a unified pattern for organizing and naming modules in Ember projects that is deterministic, extensible, and ergonomic.
#139
Ember.String.isHtmlSafe
Introduce Ember.String.isHtmlSafe() to provide a reliable way to determine if an object is an "html safe string", i.e. was it created with Ember.String.htmlSafe().
#95
Router Service
Provices a public router service.

beta

#150
Container#factoryFor
Public API to fetch a factory for creating dependency-injected instances.
